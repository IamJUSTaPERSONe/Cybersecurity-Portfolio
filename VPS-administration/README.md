# 🔐 Собственный прокси-сервер

## 🎯 Цель проекта
Развертывание защищенного прокси-сервера для выборочного направления трафика с сохранением приватности критических данных.

**Зачем:** коммерческие VPN нестабильны, собирают логи и не дают контроля над инфраструктурой.

---

## 🏗️ Архитектура

| Компонент | Что использовано |
|-----------|------------------|
| Хостинг | Aeza (Нидерланды) |
| Удаленное администрирование | PuTTY (SSH) |
| Прокси-сервер | Xray-core |
| Панель управления | 3x-ui |
| Клиент | v2rayTun / v2rayNG (Android) |
| Протокол | VLESS + Reality |

**Схема трафика:**
- Выбранные приложения/домены → через сервер
- Банки, госуслуги, поиск → напрямую (локальный IP)

---

## 🛡️ Что сделано

### Безопасность личного кабинета хостинга
- Смена дефолтного пароля
- 2FA для входа
- Привязка к Telegram

### Безопасность сервера
- Проверка конфигурации на безопасность выставленных настроек
- Смена пароля root на сложный
- Настройка фаервола (открыты только необходимые порты)
- Установка fail2ban для предотвращения автоматизированных атак

### Безопасность панели управления
- Смена дефолтных путей к панели (3x-ui) и подпискам
- 2FA при входе
- Смена логина/пароля администратора
- Смена стандартного порта на случайный порт в диапозоне 10000 - 65000 с целью снизить сканирование ботами
- Включение HTTPS соединения для предотвращения MitM атаки 

---

## 📚 Стек технологий
- **Linux (Ubuntu)** — ОС сервера
- **PuTTY** — SSH-клиент для администрирования
- **Xray-core** — прокси-сервер
- **3x-ui** — веб-панель управления
- **VLESS + Reality** — протокол с маскировкой трафика
- **uTLS** — имитация браузера
- **v2rayTun / v2rayNG** — клиент для Android
- **fail2ban** - утилита для блокировки подозрительных айпи

---

## 🔧 Краткий процесс настройки
1. **Аренда VPS** — выбор тарифа с достаточными характеристиками для комфортной скорости
2. **Установка 3x-ui** — через скрипт в PuTTY
3. **Установка fail2ban** - через скрипт в PuTTY
4. **Настройка inbound** — VLESS+Reality, порт 443, uTLS chrome
5. **Создание клиента** — генерация UUID, Public Key, Short ID
6. **Настройка клиента** — импорт подписки, раздельное проксирование
7. **Безопасность** — смена паролей, 2FA, скрытые пути, смена дефолтного порта

## 📈 Результаты
- ✅ Полная автономность от коммерческих VPN
- ✅ Выбранный трафик проходит через собственный сервер, остальной — напрямую через провайдера
- ✅ Реализована защита от DPI (Deep Packet Inspection)
- ✅ Сервер защищен на уровень выше среднего
